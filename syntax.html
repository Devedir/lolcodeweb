Implemented syntax:

  • HAI — Mandatory start of every program.
  • KTHXBYE — Mandatory end of every program.
  • VISIBLE [message] — Browser console output.
  • SCRIM [message] — Alert message.
  • I HAS A [variable] — Variable declaration.
  • LOL [variable] R [value] — Variable assignment.
  • NOT TACHIN MY [constant] OF [value] — Constant declaration and assignment.
  • UR [array] IZ FUL OF [elements] — Array declaration and assignment.
  • [elementID] MUST (ALSO) HAS [value] — Changing (or adding with ALSO) innerHTML value of an element with a given ID.
  • MEIK [attribute] OF [elementID] LIEK [value] — Changing the attribute value of an element with a given ID.
  • DRAW [property] OF [elementID] LIEK [value] — As above, but it's CSS styling.
  • INVISIBLE [message] — Exception throwing.
  • IM IN YR [loop name] WHILE/TIL [value] LIEK/BIGR THAN/SMOLR THAN [value] — Looping while/until a condition is met.
  • IM IN YR [loop name] UPPIN/NERFIN YR [iterator] FROM [value] BAI [value]
    — A classic for loop incrementing (UPPIN) or decrementing (NERFIN) from a value by a value until its explicitly broken.
  • IM IN YR [loop name] WATCHIN YR [variable] OF/IN [iterable]
    — Looping through the values (OF) or properties (IN) of an iterable object.
  • IM OUTTA YR [loop name] — Closing a loop body.
  • GTFO — Loop breaking.
  • NAH NEXT — Loop continuation.
  • IZ [value] (NOT) LIEK/BIGR THAN/SMOLR THAN [value] — Conditional statement.
  • YARLY — Optional start of a positive conditional block.
  • NOWAI — Start of an "else" block.
  • KTHX — End of a conditional block.

Note:
- All keywords must be written in UPPERCASE!
- Variable names must follow all JavaScript naming rules.
- [value] and [message] can be a literal ("Może by tak ", 5.0, etc.) or a variable.
